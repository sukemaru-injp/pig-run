# pig-run 🐖💨

ターミナル幅いっぱいのトラックを、ブタが左へ歩き続ける小さなアニメーション CLI。

```
|...🐖........🐖........🐖.....|
```

`--rainbow` を付けると、地面そのものがなめらかな虹のグラデーションになり、
色帯が横へ流れていきます（24bitカラー対応端末では truecolor、非対応端末では256色で表示）。
コマンドとして単体で動かせるほか、Claude Code のステータスラインに組み込んで
「エージェント実行中にブタが歩いている」状態にもできます。

## インストール

Go 1.26 以降が必要です。

```sh
# GitHub から直接インストール
go install github.com/sukemaru-injp/pig-run@latest

# リポジトリを clone してローカルにビルドする場合
git clone https://github.com/sukemaru-injp/pig-run.git
cd pig-run
go build -o pig-run .
```

`go install` で入れた場合、`$GOPATH/bin`（通常 `~/go/bin`）に PATH が通っている必要があります。
通っていなければシェルの設定に追記してください。

```sh
export PATH="$PATH:$(go env GOPATH)/bin"
```

## CLI として使う

```sh
pig-run                                  # 端いっぱいのトラックを3匹が歩く
pig-run --speed 50                       # 速く歩かせる
pig-run --rainbow                        # 虹の地面の上を歩く
pig-run --count 10                       # 10匹を等間隔で歩かせる
pig-run --speed 30 --count 5 --width 40  # トラック幅を手動指定
pig-run --farm                           # 牧場の中をブタたちが自由に歩き回る
pig-run --farm --count 20 --rainbow      # 虹の地面の牧場に20匹
```

`Ctrl+C` で終了します（カーソル表示も元に戻ります）。

### 牧場モード（`--farm`）

`--farm` を付けると、トラックの代わりに **縦8マス × ターミナル幅** の四角い牧場を描き、
その中を `count` 匹のブタが自由に歩き回ります。ブタは気まぐれに向きを変え、壁にぶつかると
跳ね返ります。地面には 🌱 がところどころ生えています（`--rainbow` で地面を虹にもできます）。

```
+----------------------------------+
|......🌱.........🌱.🌱....🌱......|
|....🌱...🌱..🌱...................|
|.🌱.......🐖....🌱................|
|.....🐖.............🌱..🐖........|
|...........🌱..🐖...🌱.........🐖.|
|.....🌱.🌱............🌱..........|
|.......🌱....🐖...................|
|......🌱....🌱................🌱..|
+----------------------------------+
```

`--once`（statusline 用）は1行が前提のため、`--farm` は無視されます。

### オプション一覧

| フラグ | デフォルト | 説明 |
| --- | --- | --- |
| `--speed` | `10` | アニメーション速度。大きいほど速い（待ち時間 = `1200/speed` ms。`10` で従来の 120ms 相当） |
| `--count` | `3` | 歩かせるブタの数。トラック上に等間隔で並び、右へ進んで端で折り返す |
| `--rainbow` | `false` | 地面を虹の色帯にし、その上をブタが歩く演出にする |
| `--width` | `0` | トラックの幅。`0` でターミナル幅いっぱい。数値を指定すると固定幅 |
| `--farm` | `false` | 縦8マスの牧場を描き、その中をブタたちが自由に歩き回る。🌱 がところどころ生える（`--once` 時は無視） |
| `--once` | `false` | 1フレームだけ出力して即終了。statusline などから繰り返し呼ぶ用（虹モードでも1行で出力） |

## Claude Code に組み込む

Claude Code は **ステータスラインの更新のたびに登録コマンドを1回実行**し、その標準出力を
画面下部に表示します。`--once` モードは現在時刻からコマ位置を決めて1フレームだけ出力するので、
更新のたびに次のコマが表示され、エージェント稼働中にブタが走っているように見えます。

`~/.claude/settings.json`（プロジェクト単位なら `.claude/settings.json`）に追記します。

```json
{
  "statusLine": {
    "type": "command",
    "command": "pig-run --once --rainbow"
  }
}
```

速度や匹数も渡せます。

```json
{
  "statusLine": {
    "type": "command",
    "command": "pig-run --once --speed 30 --count 2"
  }
}
```

> [!NOTE]
> ステータスラインはエージェントの活動（メッセージ更新など）に合わせて再描画されます。
> 入力待ちなどで更新が止まっている間はコマも止まりますが、ツール実行中などは
> 頻繁に更新されるため、ちょうど「動いている間だけ走る」挙動になります。

## 仕組みのメモ

- 各ブタのコマ位置は `pigPositions` で生成。等間隔に並べた位置を `step` ぶん左へずらし、
  端まで来たら反対側から出てくる（剰余で巻き戻す）。🐖 は2セル幅なので右隣のセルを空けて
  行幅を一定に保ち、折り返さずに流れる。
- 常駐モードは1行を `\r` で上書きしながら `step` を進めてアニメーションする。
- `--farm` モードは縦8マスの牧場（枠＋中身で `farmRows+2` 行）をカーソル制御（`\033[nA`）で
  毎フレーム再描画する。ブタは位置と向きを持ち、壁で反射しながら歩き回る。
- `--once` モードはカーソル制御をせず1行を出力するだけなので、statusline に安全に流し込める。
